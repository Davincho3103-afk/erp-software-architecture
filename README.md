
# Sistema ERP – Documentación de Arquitectura

Este proyecto presenta el diseño y modelado de un Sistema ERP (Enterprise Resource Planning) orientado a la gestión de procesos empresariales. El enfoque principal del trabajo se centra en el Módulo de Compras, permitiendo administrar productos, proveedores y órdenes de compra dentro de una plataforma centralizada.

El objetivo del sistema es integrar los procesos administrativos de la empresa en una sola solución tecnológica, facilitando el control del inventario, la gestión de proveedores y la generación de información para la toma de decisiones. De esta manera, se busca mejorar la eficiencia operativa y la organización de los datos empresariales.

La arquitectura propuesta corresponde a una aplicación web monolítica compuesta por un frontend tipo SPA desarrollado en React, un backend implementado en Java con Spring Boot y una base de datos PostgreSQL para el almacenamiento de la información.

El proyecto incluye diagramas de arquitectura y diseño, tales como el Diagrama de Contexto (C1), el Diagrama de Contenedores (C2), un Diagrama de Secuencia para el registro de productos y un Modelo Entidad-Relación (MER) del módulo de compras. Estos diagramas fueron elaborados utilizando PlantUML y el modelo C4.

Este trabajo tiene un propósito académico y busca aplicar conceptos de ingeniería de software como historias de usuario, criterios de aceptación, priorización de backlog y modelado arquitectónico.
